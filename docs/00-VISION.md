# Platform Core 願景

## 願景聲明

Platform Core 的願景，是建立一套可重複使用、可組合、可治理的 SaaS Framework，讓每個新產品不必重新發明共同基礎能力，同時保有專案層的獨立性。

Framework 是平台標準，不是特定客戶、品牌或產品的程式集合。只有具備明確邊界、可獨立驗證且具跨專案價值的能力，才適合成為 Platform Core 的一部分。

## 核心方向

### 可重複使用 SaaS Framework

共同能力應被定義為可替換、可組合的模組。新專案以選用與擴充模組為主，不以複製既有專案為起點。

### AI First Development

AI 參與需求分析、架構檢查、文件維護、測試設計與程式審查；所有產出仍須遵循模組邊界、變更審查與人工驗收。

### Cloudflare First

平台優先採用 Cloudflare 的全球邊緣運算與託管服務，但服務選擇必須符合責任邊界，不因平台偏好而濫用單一產品。

### Multi Tenant

從身份、權限、資料、設定到稽核，所有平台能力都必須明確處理租戶邊界，避免資料與權限跨租戶外洩。

### Multi Country

平台設計需預留語言、時區、地區格式、法規、資料保存位置與在地登入通路的差異，不將單一國家的假設寫入 Core。

### Multi Channel

平台能力應能被 Web、App、LINE、WhatsApp、WeChat、Email 與其他未來通路共用；通路只負責互動適配，不應成為核心身份或商業資料的唯一來源。

## 成功定義

- 新專案可從一致的核心文件與模組契約開始。
- 共用能力能獨立測試、版本化與替換。
- 客戶專屬需求停留在專案層，不污染 Core。
- 平台能在多租戶、多國家、多通路情境下維持一致的身份與權限邊界。
- AI 與人員使用相同的文件、規則與驗收標準協作。
