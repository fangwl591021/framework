# Cloudflare Standard

## 目的

本文件定義 Cloudflare 服務在 Platform Core 中的責任。實際專案只選用需要的服務，不以單一 Worker 承擔所有能力，也不因可用而預設導入全部服務。

## 服務責任

### Workers

執行無伺服器 API、邊緣邏輯、Adapter 與輕量協調。Worker 應保持薄層，將領域責任交給清楚分離的模組。

### D1

保存需要關聯查詢、交易一致性與結構化管理的資料。資料表應依領域與租戶邊界設計，不作為任意 JSON 儲存區。

### KV

保存高讀取、低頻更新、允許最終一致性的設定、索引或快取資料。不得用於需要即時強一致的餘額、交易或權限真相。

### R2

保存圖片、文件、匯出檔與其他大型物件。資料庫只保存物件識別、中繼資料與存取政策，不直接塞入大型二進位內容。

### Queues

處理可非同步執行、可重試與需要削峰的工作，例如通知、媒體處理或事件後續流程。Consumer 必須具備冪等性與失敗處理。

### Cron Triggers

啟動固定週期的維護、彙整或排程工作。Cron 只負責觸發，不應把大量領域邏輯寫在排程入口。

### Durable Objects

處理需要單一協調者、強一致狀態、即時連線或資源鎖定的場景。不得作為所有資料的預設儲存方案。

### AI Gateway

統一管理 AI 模型請求的觀測、供應商路由、成本、快取與安全政策。模型呼叫仍須經 AI Engine 的應用層規則管理。

### Cache

降低重複計算與來源負載，改善公開或可安全共用內容的回應速度。快取鍵必須納入會影響結果的租戶、語言、版本與權限因素。

## 選型原則

| 需求 | 優先服務 | 不適合用途 |
| --- | --- | --- |
| API 與邊緣執行 | Workers | 集中所有領域邏輯的巨型檔案 |
| 關聯與交易資料 | D1 | 大型檔案儲存 |
| 高讀取設定或快取 | KV | 強一致交易帳本 |
| 大型媒體與文件 | R2 | 權限或交易真相 |
| 非同步工作 | Queues | 需要立即同步回覆的流程 |
| 週期觸發 | Cron | 長時間單體批次邏輯 |
| 強一致協調 | Durable Objects | 一般靜態設定與分析查詢 |
| AI 流量治理 | AI Gateway | 取代 AI Engine 的領域政策 |
| 回應加速 | Cache | 未分租戶或未分權限的敏感資料 |

## 共通要求

- 所有資源命名須能辨識環境與用途。
- Production、Staging 與 Development 資源必須隔離。
- Secret 只透過受管理的 Secret 機制提供，不寫入 Repository。
- 每項服務需定義容量、錯誤、重試、監控與成本界線。
- 部署前必須確認綁定指向正確帳號、環境與資源。
